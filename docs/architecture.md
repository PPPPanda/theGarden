## 2. 核心系统

### 2.1 网格背包系统（Grid Inventory）

**概念：** 10×10 的二维网格，玩家将物品放置在网格中。物品有不同尺寸（1×1 / 2×1 / 1×2 / 2×2），摆放位置影响触发效果。

**核心规则：**
- 网格坐标原点在左下角 `(0,0)`
- 物品不可重叠，但可自由拖拽移动
- 相邻物品可触发空间协同效果（adjacency synergy）
- 网格空间是有限资源，玩家需权衡物品数量 vs 物品质量

**关键类：**
- `GridManager` — 网格状态管理（放置/移除/碰撞检测/邻接查询）
- `IGridItem` — 物品基类接口
- `IGridCell` — 单元格状态

**算法要点：**
- 放置碰撞检测：遍历物品 gridSize 覆盖的所有格子，检查 occupied 状态
- 邻接查询：获取物品周围一圈格子中的其他物品 ID（去重）
- 自动整理：可选的背包重排算法（贪心/回溯）

### 2.2 战斗时间轴引擎（Battle Timeline）

**概念：** 战斗是自动进行的，基于时间轴调度。每个物品有冷却时间，冷却完成后自动触发效果。战斗持续到一方生命值归零或超时。

**时间轴机制：**
```
时间轴 (秒)
0s ──────── 3s ──────── 6s ──────── 9s ──── ... ──── 60s
  │ 物品A触发  │ 物品B触发  │ 物品A再次   │         │ 超时判定
  │ (cd=3s)    │ (cd=6s)    │ 触发        │         │
```

**状态效果调度：**
- `Charge` — 立即推进某物品的冷却进度（秒数）
- `Haste` — 冷却速度 ×2（持续 N 秒）
- `Slow` — 冷却速度 ×0.5（持续 N 秒）
- `Freeze` — 冷却暂停（持续 N 秒）
- `Poison` — 每秒造成固定伤害
- `Shield` — 吸收下一次伤害（可叠加层数）

**关键类：**
- `BattleEngine` — 时间轴调度核心（优先队列）
- `IBattleState` — 战斗状态快照
- `ITimelineEvent` — 时间轴事件

**确定性要求：**
- 使用种子随机数（`randomSeed`），确保同一快照 + 同一种子 = 相同结果
- 支持战斗回放：记录所有 `ITimelineEvent` 到 `eventLog`

### 2.3 异步 PvP 系统（Ghost Battle）

**概念：** 不需要双方同时在线。玩家挑战的是对手的"幽灵"——一份序列化的状态快照。

**流程：**
```
玩家A 整理背包 → 上传快照到云端 → 进入匹配队列
                                         ↓
玩家B 请求对战 → 服务器匹配 MMR 相近的幽灵 → 下载幽灵数据
                                         ↓
                      客户端本地演算战斗 → 上传结果 → 更新双方 MMR
```

**关键设计：**
- `IGhostData` — 幽灵数据结构（精简序列化，减少传输量）
- MMR 匹配算法：Elo Rating 变种，±200 分范围内随机匹配
- 反作弊：服务端可抽样验证战斗结果（用相同种子重放）
- 离线收益：即使不在线，幽灵被挑战后也会更新胜负记录

### 2.4 商店系统（Shop）

**概念：** 每个回合（Day）开始时刷新商店，玩家用金币购买物品放入背包。

**规则：**
- 每轮 4-5 个物品槽位
- 物品池根据 Day 解锁（早期只有 Common，后期出现 Legendary）
- 可花费金币刷新商店
- 可锁定槽位保留到下一轮

## 3. 游戏循环

```
┌─→ 商店阶段 → 整理背包 → 匹配对手 → 自动战斗 → 结算 ─┐
│                                                      │
└──────────── Day +1, 金币结算, 循环 ←─────────────────┘
```

每个 Day 约 30-60 秒（商店 + 整理），战斗演算 5-15 秒。

## 4. 技术约束（微信小游戏）

| 约束项 | 限制 | 应对策略 |
|--------|------|----------|
| 首包大小 | 4MB | 核心逻辑 + 最小资源，其余走子包/CDN |
| 内存峰值 | 100-150MB | 对象池、纹理压缩、及时释放 |
| 网络协议 | HTTPS / WSS | 用微信云开发简化后端 |
| 渲染性能 | 中低端机适配 | 2D 为主，控制 DrawCall < 50 |
| 存储 | wx.setStorage 10MB | 关键数据上云，本地仅缓存 |
